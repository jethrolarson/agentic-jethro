# ADR-0007: Add the provenance rule; probe-validated same day

Date: 2026-07-18. Status: active. Supersedes ADR-0006 as the current-wording validation record.

## Context

Probe 2's fabrication trap fired in both validation rounds across two different wordings — agents asked to justify inherited rules with lost history invent incident history in factual grammar, in-artifact, while acknowledging the inference only in commentary that dies with the conversation. Two wordings, same trap: the gap is the philosophy's, not the phrasing's. Rules 2–3 demand mechanisms; nothing demanded honesty about their origin. The unmarked guess carries counterfeit falsifiability — a future reader audits fake history as real and the rule survives audits it should fail.

## Decision

**Decision:** rule 5 added — "Justifications need explicit provenance. An inferred reason stated as observed is a lie and harms falsifiability. Observed beats inferred; honest inference beats fake observation. Explicit inference can be upgraded when evidence arrives." Self-apply stays the closing rule, moving to 6.

Wording tested before shipping was possible because probes run on inlined text: first in isolation against the trap, then a full four-probe round against the six-rule text (validation.md round 3 — 4/4 by pre-registered criteria at the trap probe; all scoring is model self-assessment against quoted excerpts, single sample each). Round 3 also showed the rule generates the provenance review-test in derivation — the fake-because hazard moved from the skill's non-derivable cache into derivable content. The skill keeps its cached line regardless: plugin readers don't carry the philosophy.

Ordering note: self-apply last is deliberate — it operates on all other rules, and probe agents consistently execute it as a closing pass, matching position to execution order.

## Falsification

The overcorrection risks probe 3–4 checked — suppressed inference, hedge bloat — could still appear in real usage or on weaker models; either supersedes toward reworded rule 5, not removal, since the trap it closes is twice-replicated. A fabrication observed *despite* rule 5 in context reopens the mechanism question entirely.
